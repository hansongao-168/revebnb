# UniApp frontend (revebnb)

This directory holds **source files** aligned with the `frontend-user-auth` skill: login, Pinia user store, HTTP helpers, route guard, and profile API calls.

## Prerequisite: create a UniApp + Vue3 + Vite project

UniApp’s toolchain (`@dcloudio/vite-plugin-uni`, `manifest.json`, `index.html`) is best generated by the official scaffolder, then you merge this `src/` and root config files.

```bash
# Example (pick a template that matches your targets: H5 / 微信小程序 / App)
npm create uni@latest revebnb-uni
cd revebnb-uni
```

Copy from this repo into the generated project:

- `pages.json` (merge entries if your template already defines pages)
- `src/` (entire tree)
- `tsconfig.json` (merge `paths` for `@/*` if you already have one)
- Set `VITE_API_BASE_URL` in `.env` (see `src/config.ts`)

In `vite.config.ts`, ensure the `@` alias points at `src` (official templates usually include this):

```ts
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
})
```

## Environment

| Variable | Example |
|----------|---------|
| `VITE_API_BASE_URL` | `http://127.0.0.1:8000` (no trailing slash) |

Laravel must expose `GET/POST` under `/api/*` (Sanctum token auth). CORS is enabled for `api/*` in this Laravel app.

## WeChat login

Configure Laravel `.env`:

- `WECHAT_MINI_APP_ID`
- `WECHAT_MINI_APP_SECRET`

When unset, **only with `APP_DEBUG=true`**, the API uses a deterministic mock `openid` from `loginCode` so local H5 flows can be exercised without WeChat credentials.

## Laravel API reference

| Method | Path | Auth |
|--------|------|------|
| POST | `/api/auth/register` | No |
| POST | `/api/auth/login` | No |
| POST | `/api/auth/wechat-login` | No |
| GET | `/api/user/profile` | Bearer token |
| PUT | `/api/user/profile` | Bearer token |
| POST | `/api/user/avatar` | Bearer token (multipart `avatar`) |
