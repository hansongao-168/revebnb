# 外部日历（ICS）同步 — 运营说明

面向平台运营、客服与实施同学。技术设计见 `docs/superpowers/specs/2026-05-18-ics-calendar-sync-design.md`。

**在线阅读（无需登录）**：站点路径 `/docs/ics-external-calendar`

---

## 1. 功能做什么

把 **Airbnb 等平台的 iCal 订阅链接** 绑定到房源，系统会定期（或手动）拉取外部「已订 / 不可订」日期，并：

1. 在后台 **日历对比** 页与本地「已确认订单」「手动不可租」并列展示；
2. 可选：开启 **合并进前台可订** 后，访客在 `/stays` 预订页会看到这些天不可选。

**重要**：本系统 **不会** 把 revebnb 的订单写回 Airbnb；仅为 **单向读取** 外部日历。

---

## 2. 谁能做什么

| 角色 | 面板 | 配置 ICS / 同步 | 日历对比（只读） | 合并进前台可订 |
|------|------|----------------|------------------|----------------|
| 平台运营 | `/admin` | ✅ | ✅ | ✅（按 feed 开关） |
| 租户员工 | `/tenant-admin` | ❌ | ✅ | ❌ |
| 房东 | `/landlord-portal` | ❌ | ✅ | ❌ |

ICS 订阅 URL（含私密 token） **仅在平台后台** 录入与修改。

---

## 3. 获取 Airbnb ICS 链接 {#airbnb-ical}

1. 登录 Airbnb 房东后台，打开对应房源。
2. 进入 **日历** → **可用性设置** / **同步日历**（文案因地区而异）。
3. 导出或复制 **iCal 链接**，格式类似：  
   `https://www.airbnb.fr/calendar/ical/{房源ID}.ics?t={私密令牌}`
4. **整段 URL 只交给平台后台**，勿发到群聊、工单截图或公开文档。链接泄露等于日历可被他人读取。

同一房源若有 **Booking.com** 等，可再添加第二条 feed（不同「显示名称 / 来源标识」）。

---

## 4. 平台后台操作步骤

### 4.1 绑定订阅

1. 登录 **平台管理后台** → `/admin`
2. 侧栏 **租房** → **房源** → 打开目标房源 **编辑**
3. 下方 Relation **「外部日历」** → **新建**
4. 填写：
   - **显示名称**：如 `Airbnb 主日历`
   - **来源标识**（可选）：如 `airbnb`
   - **ICS 订阅 URL**：粘贴完整 https 链接
   - **启用同步**：建议开启
   - **同步间隔（小时）**：留空则默认 **6 小时**
   - **合并进前台可订**：默认 **关闭**；确认对比无误后再开启
5. 保存后，点击 **「立即同步」** 或等待定时任务

### 4.2 同步状态字段

| 列 | 含义 |
|----|------|
| 状态 | `success` / `failed` / 同步中 |
| 上次成功 | 最近一次成功拉取时间 |
| 事件数 | 当前库中该 feed 的外部事件条数 |

失败时查看 **状态** 旁错误摘要（不含完整 URL）。常见原因：链接失效、token 被 Airbnb 轮换、网络超时。

### 4.3 日历对比

- 编辑页顶栏 **「日历对比」**，或访问  
  `/admin/listings/{房源ID}/calendar`
- 选择月份，查看：
  - **仅外部占用** / **仅本地占用** / **重叠**
  - 当月外部事件列表

### 4.4 批量同步

- 单条 feed：行操作 **「立即同步」**
- 该房源全部已启用 feed：表头 **「同步全部」**

## 5. 合并进前台可订（谨慎开启）

对某条 feed 开启 **「合并进前台可订」** 后：

- 该 feed 同步到的占用日期会进入访客站 `/stays/{slug}` 的不可选日历；
- 访客提交订单时，若日期落在外部占用夜，会提示日期已被预订。

**建议流程**：

1. 先同步，在 **日历对比** 确认与 Airbnb 一致；
2. 再开启「合并进前台可订」；
3. 打开前台房源详情，切换月份 spot check。

关闭开关后，外部数据仍保留在后台对比中，但 **不再阻挡** 访客预订。

## 6. 租户 / 房东

- **租户** `/tenant-admin`、**房东** `/landlord-portal`：编辑房源 → **「日历对比」**
- 仅查看，页面会提示：ICS 由平台配置
- 无法新增、修改 URL 或触发同步（需联系平台运营）

## 7. 服务器与定时任务（给运维）

| 项 | 说明 |
|----|------|
| 数据库迁移 | 部署后执行 `php artisan migrate` |
| 定时任务 | Cron 需每分钟执行 `php artisan schedule:run`；系统已注册 **每小时** `calendar-feeds:sync-due`（扫描到期 feed 并入队） |
| 队列 | 同步走队列 Job；生产需运行 `php artisan queue:work`（或等价 Supervisor 配置）。本地 `composer run dev` 已含 queue |
| 手动命令 | `php artisan calendar-feeds:sync-due`（仅派发到期 feed 的 Job） |

### 环境变量（可选）

| 变量 | 默认 | 说明 |
|------|------|------|
| `CALENDAR_FEED_DEFAULT_SYNC_INTERVAL_HOURS` | `6` | 未单独设置间隔的 feed 使用该值 |
| `CALENDAR_FEED_EMPTY_ICS_CLEARS` | `true` | 拉取成功但 ICS 无事件时，是否清空该 feed 已有事件 |
| `CALENDAR_FEED_ALLOW_HTTP` | `false` | 仅本地测试可开；生产应禁止 http ICS |
| `CALENDAR_FEED_HTTP_TIMEOUT` | `30` | HTTP 超时（秒） |

## 8. 常见问题

### Q：同步失败，提示 HTTP 401/404

Airbnb 链接中的 `?t=` token 已失效或被房东重新导出。请在 Airbnb 重新复制 **新的** iCal 链接，在平台后台 **编辑** 该 feed 更新 URL（留空则不修改 URL 的字段请重新粘贴完整新链接）。

### Q：对比页有外部日期，但前台仍能选

检查该 feed 是否开启 **「合并进前台可订」**。未开启时仅后台可见。

### Q：Airbnb 有订，但对比页没有

1. 是否 **启用同步** 且 **立即同步** 成功；
2. 查看 **事件数** 是否为 0；
3. 是否看错月份（对比页按 UTC/应用时区切日）；
4. 是否绑错房源的 ICS 链接。

### Q：同步成功但事件变少了

全量对账：ICS 里已消失的订单会从本系统删除。若 Airbnb 暂时返回空日历且 `CALENDAR_FEED_EMPTY_ICS_CLEARS=true`，会清空该 feed 全部事件（默认行为）。

### Q：能否在房东面板改链接

不能。统一由平台 `/admin` 管理，避免 token 分散与权限混乱。

---

## 9. 安全提醒

- ICS URL 含 **私密令牌**，数据库中 **加密存储**，界面不展示完整链接。
- 勿将链接写入公开工单、邮件正文或截图。
- 离职运营无需单独清理 token，但应在 Airbnb 侧 **重置导出链接** 若怀疑泄露。
